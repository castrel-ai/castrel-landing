## What Makes Castrel Different?

- **Context-aware without CMDB complexity** — Castrel learns how to gather and correlate context from your infrastructure. Through simple prompts or documentation you provide, it understands how to fetch the right data. No need for extensive CMDB setup or painful data cleansing.

- **Runbook-driven execution without workflow builders** — Skip the drag-and-drop workflow configuration. Castrel executes inspection, troubleshooting, and remediation tasks based on documents. Auto-generate them, upload your existing ones, or edit inline — it just works.

- **Grounded reasoning with full transparency** — Every step of Castrel's analysis is backed by actual data. No hallucinations. Every query executed is logged, and you can inspect the supporting evidence at any time.

## Core Capabilities

- **Alert Triage** — Automatically classify alerts as False Positive, Potential Risk, or Incident by correlating metrics, logs, and context.
- **Incident Investigation** — Generate hypotheses, gather evidence, and collaborate with you to pinpoint root causes faster.
- **Task Execution** — Plan and execute operational tasks with runbook-based workflows and human confirmation for sensitive operations.
- **System Q&A** — Query your system  using natural language. Mention services / IT resources with @ to ask about status, architecture, or alerts.

## Interaction Model

Castrel meets you where you work:

- **Slack** — Triage alerts and get analysis without leaving your chat
- **Web UI** — Deep-dive into investigations with detail
- **Local Agent Proxy** — Connect local diagnostics to Castrel's intelligence for file analysis and command execution
