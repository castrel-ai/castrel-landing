## What is Castrel Proxy?

Castrel Proxy is a lightweight local proxy that bridges your local environment with Castrel's AI capabilities. It enables Castrel to directly access diagnostic data on your local machine—files, command outputs, and system information—without requiring you to manually copy-paste or upload anything.

This solves a common challenge in troubleshooting: **critical diagnostic data often exists only in local environments** and cannot be accessed through cloud integrations. With Castrel Proxy, you get "local data + cloud intelligence" working seamlessly together.

## How it Works

**Connection Architecture**

Castrel Proxy establishes a secure WebSocket connection between your local machine and Castrel's backend:

```mermaid
flowchart LR
    A[Local Proxy] -->|WebSocket/TLS| B[Castrel Agent]
```

The connection is:
- **Outbound-only**: Local Proxy initiates all connections to Castrel Agent (no inbound ports required)
- **Encrypted**: All traffic uses TLS encryption
- **Authenticated**: Requires workspace verification code + user token
- **User-controlled**: Can be terminated anytime from either the cloud console or the local client


**Two-Layer Concept: Connection vs Activation**

| Concept | Scope | Description |
| --- | --- | --- |
| **Connection** | User-level (global) | The outbound WebSocket connection from Local Proxy to Castrel Agent. Visible across all chat sessions. |
| **Activation** | Session-level | Binding a connection to a specific chat session. AI can only use activated connections. |

::callout{icon="i-lucide-info" color="info"}
**Key Rules**:
- Each session can activate at most **one** connection
- Activating a different connection automatically deactivates the previous one
- Disconnecting a connection deactivates it from all sessions
::


## Getting Started

**Installing the Proxy Agent**

Install the Castrel Proxy agent with a single command:

```bash
pip install castrel-proxy
```

::callout{icon="i-lucide-info" color="info"}
**Requirement**: Python >= 3.10
::

**Establishing a Connection**

1. In any Castrel chat session, click the **Local Proxy button** (📡) in the input area footer
2. Click **+ New Connection** to generate a verification code
3. Run the connection command in your local terminal:

```bash
castrel-bridge-cli pair <VERIFICATION_CODE> <API_URL> && castrel-bridge-cli start
```

4. Once connected, click **Activate to Current Session** to enable AI access

::callout{icon="i-lucide-lightbulb" color="primary"}
**Tip**: The verification code expires in 5 minutes. If it expires, simply generate a new one.
::

**Connection Status Indicators**

The Local Proxy button shows different states:

| Status | Indicator | Meaning |
| --- | --- | --- |
| Inactive | Gray icon | No connection activated in current session |
| Active | Green icon with dot | Connection activated and ready |
| Pending | Yellow pulsing | Waiting for proxy agent to connect |
| Disconnected | Gray icon with red dot | Previously activated connection lost |

## Core Capabilities

**Local File Analysis**

Request analysis of local files using natural language:

```
User: Analyze /tmp/heapdump.hprof and find memory leak points

AI: I'll analyze the heap dump file on your local machine...
    [Streams and analyzes the 2GB file]

    Found 3 potential memory leak points:
    1. SessionCache holding 2.8M expired entries (1.2GB)
    ...
```

**Supported file types**:
- Text files: logs, configs, scripts
- Java Heap Dumps: `.hprof` format
- Thread Dumps: jstack output
- Core Dumps: basic information extraction

**Large file handling**:
- Files under 10MB: Complete transfer then analysis
- Files over 10MB: Streaming analysis with key information extraction

**Real-time Command Execution**

Execute diagnostic commands through natural language:

```
User: Check the current process status on my machine

AI: I'll run top to check process status on Alex-MacBook-Pro...
    [Executes: top -l 1 -n 20]

    Current system: CPU 45%, Memory 72%
    Top process: java (PID 12345) using 4.2GB memory
    ...
```

## Security Controls

**Execution Modes**

| Mode | Description | Behavior |
| --- | --- | --- |
| **Safe Mode** (default) | Commands not on whitelist require confirmation | Recommended for most users |
| **Auto-run Mode** | All commands execute automatically | For advanced users, emergency scenarios |

::callout{icon="i-lucide-shield-alert" color="warning"}
Switching to Auto-run Mode requires explicit confirmation. Blacklisted dangerous commands are **always blocked** regardless of mode.
::

## When to Use Castrel Proxy

**Ideal For:**

- Analyzing large diagnostic files (heap dumps, core dumps)
- Accessing logs not integrated with cloud observability
- Running real-time system diagnostics
- Executing custom diagnostic scripts
- Troubleshooting local development environments

**Not Ideal For:**

- Data already available through Connectors (use Connectors instead)
- Automated/scheduled tasks (use proper CI/CD tools)
- Persistent monitoring (set up proper observability stack)

## Common Questions

::collapsible{name="Can I connect multiple devices?"}
Yes. You can establish connections from multiple devices simultaneously. Each chat session can activate one connection at a time, but you can easily switch between devices using the connection management panel.
::

::collapsible{name="Is my data transmitted to Castrel servers?"}
File contents and command outputs are transmitted over encrypted connections for AI analysis. The data is processed in memory and not permanently stored. For sensitive environments, review your organization's data policies before use.
::

::collapsible{name="Can I use Castrel Proxy in air-gapped environments?"}
The proxy requires network connectivity to Castrel's cloud services. For air-gapped environments, contact our enterprise team for on-premises deployment options.
::

::collapsible{name="What permissions does the proxy agent need?"}
The proxy agent runs with your user permissions. It can only access files and execute commands that your user account has permission to access. No elevated privileges are required for installation or operation.
::

::collapsible{name="How do I uninstall the proxy agent?"}
Simply delete the binary from your installation path (typically `/usr/local/bin/castrel-bridge-cli` on macOS/Linux). No background services or system modifications are made.
::
