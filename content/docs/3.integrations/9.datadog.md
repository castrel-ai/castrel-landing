# Datadog Connector Instructions

The Datadog Connector enables seamless integration with Datadog, allowing Castrel to query and analyze Datadog's logs, metrics, monitors, and events.

## 📚 Overview

- **Connector Name**: `datadog`
- **Supported API Version**: Datadog REST API v1/v2
- **Supported Server Versions**: Datadog SaaS (all regions)
- **Official Documentation**: [Datadog API Docs](https://docs.datadoghq.com/api/latest/)

## ⚙️ Configuration

To use the Datadog tools, you need to configure a Connector with the following fields:

| Field | Required | Description |
|-------|----------|-------------|
| `instance_name` | Yes | Instance name to identify this connector instance. |
| `base_url` | Yes | The base URL of your Datadog API (e.g., `https://api.datadoghq.com`). |
| `api_key` | Yes | DD-API-KEY for account identification. |
| `app_key` | Yes | DD-APPLICATION-KEY for read access. |
| `verify_ssl` | No | SSL certificate verification (default: true). |

## 🛠 Available Tools

The connector provides tools for logs, metrics, monitors, and events exploration.

- **Log Search**: Search and aggregate logs with query syntax.
- **Metric Query**: Query timeseries data, list metrics, and get metadata.
- **Monitor Search**: Search monitors and get monitor details.
- **Event Search**: Get and search events.
