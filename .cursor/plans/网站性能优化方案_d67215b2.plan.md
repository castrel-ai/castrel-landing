# Castrel.ai 网站性能优化方案（非视频部分）

## 当前问题总结

| 指标 | 当前值 | 目标值 | 状态 |

|------|--------|--------|------|

| FCP | 2.78s | <1.8s | 需优化 |

| LCP | 4.12s | <2.5s | 严重超标 |

| TTFB | 1.87s | <0.8s | 较慢 |

---

## 第一阶段：图片优化

### 1.1 集成 @nuxt/image 模块

在 [`nuxt.config.ts`](nuxt.config.ts) 中添加配置：

```typescript
modules: ['@nuxt/image'],
image: {
  provider: 'vercel',
  domains: ['castrel.ai'],
  formats: ['avif', 'webp'],
  quality: 80,
}
```

### 1.2 替换原生 `<img>` 为 `<NuxtImg>`

需要修改的组件：

- [`app/components/IncidentImages.vue`](app/components/IncidentImages.vue) - incident1.png, incident2.png
- [`app/components/TaskImage.vue`](app/components/TaskImage.vue) - tasks.png
- [`app/components/ProductFeatureTab.vue`](app/components/ProductFeatureTab.vue) - 动态图片
- [`app/components/IntegrationGrid.vue`](app/components/IntegrationGrid.vue) - 集成图标
- [`app/components/BlogPostCard.vue`](app/components/BlogPostCard.vue) - 博客封面
- [`app/pages/blogs/[...slug].vue`](app/pages/blogs/[...slug].vue) - 博客详情页图片

### 1.3 批量压缩现有大图片

优先处理大文件（使用 Squoosh 或 Sharp）：

- `blog/ai-troubleshooting/incident-investigation-header.png` (7.1MB)
- `blog/ai-troubleshooting/knowledge-accumulation.png` (5.3MB)
- `blog/ai-troubleshooting/ai-analysis.png` (5.0MB)
- `slack/slack-demo.gif` (1.2MB) - 考虑转为视频或优化

## 第二阶段：字体优化

### 2.1 添加关键字体预加载

在 [`nuxt.config.ts`](nuxt.config.ts) 的 `app.head.link` 中添加：

```typescript
{ rel: 'preload', href: '/fonts/ia-writer-quattro/ia-writer-quattro-400.woff2', as: 'font', type: 'font/woff2', crossorigin: '' },
{ rel: 'preload', href: '/fonts/fira-code/fira-code-400.woff2', as: 'font', type: 'font/woff2', crossorigin: '' },
```

### 2.2 确保 font-display: swap

在 [`app/assets/css/fonts.css`](app/assets/css/fonts.css) 中检查并确保所有 `@font-face` 都有 `font-display: swap`。

---

## 第三阶段：Vercel 缓存配置

创建 [`vercel.json`](vercel.json)：

```json
{
  "headers": [
    {
      "source": "/images/(.*)",
      "headers": [{ "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }]
    },
    {
      "source": "/fonts/(.*)",
      "headers": [{ "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }]
    },
    {
      "source": "/videos/(.*)",
      "headers": [{ "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }]
    }
  ]
}
```

---

## 第四阶段：Nuxt/Nitro 配置优化

在 [`nuxt.config.ts`](nuxt.config.ts) 中增强 Nitro 配置：

```typescript
nitro: {
  prerender: {
    crawlLinks: true,
    routes: ['/'],
  },
  compressPublicAssets: true,
}
```

优化 Vite 构建配置，将 mermaid 单独打包：

```typescript
vite: {
  build: {
    cssCodeSplit: true,
    rollupOptions: {
      output: {
        manualChunks: {
          mermaid: ['mermaid'],
        }
      }
    }
  }
}
```

## 第五阶段：代码优化

### 5.1 懒加载重型组件

对 mermaid 相关组件使用 `defineAsyncComponent` 异步加载。

### 5.2 按需加载 Speed Insights

确保 `@vercel/speed-insights` 仅在生产环境加载。

## 预期效果

| 指标 | 当前值 | 优化后预期 | 提升 |

|------|--------|------------|------|

| FCP | 2.78s | 1.2-1.5s | -45-55% |

| LCP | 4.12s | 1.8-2.2s | -55-60% |

| TTFB | 1.87s | 0.5-0.8s | -60-70% |

---

## 实施优先级

1. **高优先级**：图片优化（对 LCP 影响最大）
2. **中优先级**：Vercel 缓存配置、Nitro 预渲染
3. **低优先级**：字体预加载、代码懒加载
